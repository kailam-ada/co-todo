Co-Todo · Brand & UI Kit · Conforme RGAA 4.1.2 · 100/100 · v1.1 · Mai 2026

Une charte chaleureuse pour deux co‑parents qui s'organisent.

Co-Todo accompagne les familles séparées dans le quotidien : tâches partagées, bonus de planification, rappels apaisés. La charte est claire, douce, profondément accessible — pensée d'abord pour les écrans mobiles et pour la fatigue mentale.

01 — Fondations

Principes de marque

Quatre principes pilotent chaque décision visuelle. Quand un choix nous fait douter, on revient à cette liste.

Chaleureux, jamais clinique

Crème, terre cuite, sauge. On évite les blancs cliniques et les bleus hôpital.

Coopération visible

Deux formes qui se superposent — la marque dit « ensemble » avant tout texte.

Structurel et lisible

Hiérarchie nette, corps ≥ 16 px, interligne 1.6, contraste AAA pour le texte.

Jamais la couleur seule

Chaque signal couleur est doublé d'une icône, d'un motif ou d'un texte explicite.

02 — Couleurs

Palette accessible

Une palette restreinte : 1 primaire pour les actions, 1 accent pour le bonus de planification, 2 signaux (succès / destructeur) et une gamme de gris chauds. Les ratios sont calculés en direct ci-dessous (algorithme WCAG 2.x — luminance relative).

03 — Contrastes

Matrice de validation

Référentiel : 4.5:1 pour le corps de texte, 3:1 pour les grands titres (≥ 24 px ou ≥ 19 px gras), 3:1 pour les éléments graphiques porteurs d'information (icônes, bordures, focus). Calculs effectués selon WCAG 2.2 §1.4.3 / §1.4.11.

04 — Signaux non‑chromatiques

L'information jamais portée par la couleur seule

Critère RGAA 3.1. Chaque état est codé sur deux ou trois canaux indépendants : couleur + icône, couleur + motif, ou couleur + texte explicite. Un utilisateur daltonien ou en mode niveaux de gris distingue chaque signal sans ambiguïté.

05 — Typographie

06 — Boutons

États interactifs

Quatre variantes (primaire, accent, destructeur, fantôme) × quatre états (par défaut, survol, focus clavier, actif). Cible tactile ≥ 44 × 44 px (WCAG 2.5.5). L'anneau de focus est universel : 3 px solide, offset 2 px, bleu focus à 6.4:1 sur fond crème — volontairement distinct de toutes les couleurs sémantiques.

Journal des corrections appliquées

Les six points relevés lors du pré‑audit ont été corrigés dans la version 1.1 du kit. Détail des correctifs :

A‑01Résolu ✓
Critère 7.1 · WAI‑ARIA Authoring Practices · Tab Pattern

Onglets conformes au pattern ARIA

Les onglets exposent role="tablist"/tab + aria-selected, avec roving tabindex et navigation clavier ← → Home End. Les groupes segmentés bénéficient de la même gestion Flèches.

Correctif : composant Tabs reconstruit dans co-todo-kit.js (sélection au clic et au clavier, focus géré).

A‑03Résolu ✓
Critère 9.1 · WCAG 1.3.1 · Info and Relationships

Hiérarchie de titres sans saut de niveau

Les sous‑titres des cartes (principes de marque, jauges, mocks) passent de h4 à h3 : l'arbre de titres enchaîne h1 → h2 → h3 sans rupture.

Correctif : balises promues en h3, sélecteurs CSS associés mis à jour.

A‑02Résolu ✓
Critère 7.5 · WCAG 4.1.3 · Status Messages

Résultats dynamiques annoncés

La zone de résultats recalculés porte aria-live="polite" + aria-atomic="true" ; tout changement est vocalisé par NVDA / VoiceOver.

Correctif : attributs ajoutés sur la région #btn-states.

A‑04Résolu ✓
Critère 11.5 · WCAG 1.3.1

Groupes de champs regroupés

Les groupes de cases (« Choix de répartition ») et de radios (« Répartition ») sont encapsulés dans <fieldset> + <legend>.

Correctif : fieldset/legend ajoutés aux § 07 et § 08.

A‑05Résolu ✓
Critère 11.13 · WCAG 1.3.5 · Identify Input Purpose

Finalité des champs identifiée

Le champ e‑mail porte type="email" + autocomplete="email" (liste fermée WCAG 1.3.5).

Correctif : attributs type et autocomplete renseignés.

A‑06Résolu ✓
Critère 5.6 / 5.7 / 8.9

Synthèse en table HTML native

La synthèse par thématique est désormais une <table> : <caption>, <thead>, <th scope="col"> et <th scope="row">.

Correctif : div[role=table] remplacé par <table><thead><tbody>.
